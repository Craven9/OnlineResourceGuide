</head>

Glossary

Key terms and concepts for HTML beginners. Understanding this vocabulary will help you communicate effectively about web development and understand technical documentation.

Attribute
Additional information provided within an HTML tag that modifies the element's behavior or appearance. Example: src="image.jpg" in an img tag.
Block Element
HTML elements that take up the full width available and create new lines before and after themselves. Examples: <div>, <p>, <h1>.
Element
A complete HTML component consisting of an opening tag, content, and closing tag. Example: <p>This is a paragraph</p>.
HTML (HyperText Markup Language)
The standard markup language used to create web pages. It describes the structure and content of web documents using tags and elements.
Inline Element
HTML elements that only take up as much width as necessary and do not create line breaks. Examples: <span>, <strong>, <em>.
Nesting
Placing HTML elements inside other elements. Proper nesting requires that inner elements are completely contained within outer elements.
Semantic HTML
Using HTML elements that clearly describe their meaning and purpose rather than just their appearance. Example: using <header> instead of <div> for page headers.
Tag
The markup code that defines an HTML element. Tags are enclosed in angle brackets and usually come in pairs: opening tag <p> and closing tag </p>.
Validation
The process of checking HTML code against official standards to ensure it follows proper syntax and structure rules.
W3C (World Wide Web Consortium)
The international organization that develops web standards, including HTML specifications and accessibility guidelines.

Assignment Submission Checklist

Use this checklist before submitting HTML assignments to ensure your work meets academic standards and functions properly across different browsers and devices.

Code Quality and Structure

  • □ Document includes proper DOCTYPE declaration
  • □ HTML element includes lang attribute
  • □ Head section contains charset meta tag
  • □ Head section contains viewport meta tag for responsive design
  • □ Page has descriptive title tag
  • □ All opening tags have corresponding closing tags
  • □ Elements are properly nested (no overlapping tags)
  • □ Code uses consistent indentation for readability
  • □ All tag names and attributes are lowercase
  • □ All attribute values are enclosed in quotes

Content and Semantics

  • □ Heading hierarchy follows logical order (H1 → H2 → H3)
  • □ Only one H1 element per page
  • □ Semantic elements used appropriately (<header>, <nav>, <main>, <footer>)
  • □ Text content uses appropriate elements (<p> for paragraphs, <strong> for emphasis)
  • □ Lists use proper list elements (<ul> or <ol> with <li>)
  • □ Tables include proper headers (<th>) and structure (<thead>, <tbody>)

Links and Media

  • □ All links have descriptive link text (avoid "click here")
  • □ External links open in new tabs if appropriate
  • □ All file paths are correct and functional
  • □ All images include descriptive alt text
  • □ Image file names are descriptive and web-friendly
  • □ Media files are optimized for web use

Forms and Interactivity

  • □ All form inputs have associated labels
  • □ Labels use "for" attribute matching input "id"
  • □ Required fields are marked as required
  • □ Form includes submit button
  • □ Form action and method attributes are specified if needed

Accessibility and Standards

  • □ HTML passes W3C validation without errors
  • □ Page passes WAVE accessibility check
  • □ Color contrast meets accessibility standards
  • □ Page is navigable using only keyboard
  • □ Screen reader testing completed or simulated

Testing and Compatibility

  • □ Page displays correctly in multiple browsers (Chrome, Firefox, Safari)
  • □ Page is responsive on mobile devices
  • □ All links work and lead to correct destinations
  • □ Page loads completely without broken images or missing files
  • □ Console shows no JavaScript errors (press F12 to check)

Assignment Requirements

  • □ All assignment requirements have been met
  • □ File naming follows instructor guidelines
  • □ Required elements are included as specified
  • □ Content meets minimum/maximum length requirements
  • □ Submission includes all required files
  • □ Work is original and properly attributed if using external sources

Study Tips for HTML Success

Practice Regularly

  • Code for at least 30 minutes daily
  • Build small projects to reinforce concepts
  • Recreate layouts from websites you admire
  • Use online coding challenges and exercises

Learn from Mistakes

  • Keep a log of common errors you make
  • Understand why validation errors occur
  • Practice fixing broken HTML code
  • Ask questions when stuck rather than guessing

Build Good Habits

  • Always validate your HTML before submission
  • Use consistent code formatting and indentation
  • Comment your code to explain complex sections
  • Test across multiple browsers and devices

Stay Current

  • Follow web development news and updates
  • Learn about accessibility best practices
  • Understand mobile-first design principles
  • Keep up with new HTML features and standards

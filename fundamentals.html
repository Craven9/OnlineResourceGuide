HTML Document Structure

Every HTML document has this basic structure:

<!DOCTYPE html>           
<html lang="en">             
<head>                       
    <meta charset="UTF-8">    
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title> 
</head>
<body>                       
    
</body>
</html>

HTML Syntax Rules

Tags

HTML tags are enclosed in angle brackets: <tagname>

<h1>Heading</h1>
<p>Paragraph</p>
<br> <!-- Self-closing tag -->

Attributes

Provide additional information about elements

<a href="https://example.com">Link</a>
<img src="image.jpg" alt="Description">
<div class="container" id="main"></div>

Nesting

Elements can contain other elements

<div>
    <h2>Section Title</h2>
    <p>Paragraph with <strong>bold</strong> text</p>
</div>

Common HTML Elements

Text Elements

<h1>Main Heading</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<p>This is a paragraph</p>

Lists

Links and Images

<!-- Links -->
<a href="https://example.com">External Link</a>
<a href="page.html">Internal Link</a>
<a href="#section">Anchor Link</a>

<!-- Images -->
<img src="image.jpg" alt="Description">
<img src="photo.png" alt="Photo" width="300" height="200">

HTML Best Practices & Style Guide

Visual guide to HTML coding standards

  • Use Semantic HTMLChoose tags based on meaning, not appearance. Use <header>, <nav>, <main>, <article>, and <footer> appropriately.
  • Always Close TagsProperly nest and close all elements to ensure valid HTML structure and avoid rendering issues.
  • Use LowercaseWrite all tags and attributes in lowercase for consistency and better readability.
  • Quote AttributesAlways use quotes around attribute values, even for single words, to maintain consistency.
  • Add Alt TextInclude descriptive alt text for all images to improve accessibility for screen readers.
  • Validate Your CodeUse HTML validators to check for errors and ensure your code follows web standards.

HTML Quick Reference

Basic Page Structure

Every HTML document follows a standard structure that tells the browser how to interpret and display your content. This foundation ensures your pages work correctly across different browsers and devices.

Essential Document Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Your Page Title Here</title>
</head>
<body>
    <!-- Your visible content goes here -->
</body>
</html>

Structure Elements Explained

  • <!DOCTYPE html> - Tells browser this is HTML5 (required first line)
  • <html lang="en"> - Root element with language declaration for accessibility
  • <head> - Contains metadata not visible to users
  • <meta charset="UTF-8"> - Character encoding for international text support
  • <meta name="viewport"> - Responsive design setup for mobile devices
  • <title> - Page title shown in browser tab and search results
  • <body> - Contains all visible page content

Text and Structure Tags

Text elements provide the foundation for organizing and formatting content. Understanding proper heading hierarchy and text formatting ensures both readability and accessibility compliance for assignments.

Headings (H1-H6)

<h1>Main Page Title</h1>
<h2>Major Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Heading Best Practices

  • Use only one H1 per page for the main title
  • Do not skip heading levels (H1 → H3 without H2)
  • Use headings for structure, not just visual formatting

Paragraphs and Text Formatting

<p>This is a paragraph. Use paragraphs to group related sentences.</p>

<p>Text can include <strong>important text</strong> and <em>emphasized text</em>.</p>

<p>You can also use <code>inline code</code> for technical terms.</p>

<blockquote>
    This is a longer quote or excerpt from another source.
    Use blockquote for citations or notable text passages.
</blockquote>

Lists

Unordered Lists (Bullet Points)

<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ul>

Ordered Lists (Numbered)

<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

Definition Lists

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>

Links and Images

Links and images connect your content to the broader web and provide visual elements. Proper implementation includes accessibility features and clear navigation patterns essential for user experience.

Links

<!-- External link (to another website) -->
<a href="https://www.example.com">Visit Example Website</a>

<!-- Internal link (to another page on your site) -->
<a href="about.html">About Page</a>

<!-- Anchor link (to section on same page) -->
<a href="#contact-section">Jump to Contact Section</a>

<!-- Email link -->
<a href="mailto:student@example.com">Send Email</a>

Understanding File Paths

  • Relative path: about.html (same folder)
  • Relative path: pages/about.html (in subfolder)
  • Relative path: ../index.html (up one folder)
  • Absolute path: https://www.example.com/page.html (full URL)

Images

<!-- Basic image -->
<img src="photo.jpg" alt="Description of the image for screen readers">

<!-- Image with size specifications -->
<img src="logo.png" alt="Company logo" width="200" height="100">

<!-- Image as a link -->
<a href="full-size.jpg">
    <img src="thumbnail.jpg" alt="Click to view full size image">
</a>

Image Best Practices

  • Always include alt text describing the image content
  • Use descriptive file names (not IMG_001.jpg)
  • Optimize image file sizes for web loading
  • Use appropriate file formats: JPG for photos, PNG for graphics with transparency

Semantic Layout Elements

Semantic HTML elements provide meaningful structure that improves accessibility, search engine optimization, and code maintainability. These elements clearly identify different content sections and their purposes.

<header>
    <h1>Website Title</h1>
    <nav>
        <ul>
            <li><a href="/">Home</a></li>
            <li><a href="/about">About</a></li>
            <li><a href="/contact">Contact</a></li>
        </ul>
    </nav>
</header>

<main>
    <article>
        <h2>Article Title</h2>
        <p>Main article content goes here.</p>
    </article>
    
    <aside>
        <h3>Related Links</h3>
        <p>Sidebar content goes here.</p>
    </aside>
</main>

<footer>
    <p>Copyright 2025 Website Name</p>
</footer>

Semantic Elements Explained

  • <header> - Page or section header with titles and navigation
  • <nav> - Navigation links and menus
  • <main> - Primary content of the page (use only once per page)
  • <article> - Self-contained content that could stand alone
  • <section> - Thematic grouping of content with heading
  • <aside> - Sidebar content related but not essential to main content
  • <footer> - Page or section footer with metadata

Tables and Forms Basics

Tables display tabular data clearly, while forms collect user input. Both require proper structure and labeling for accessibility and usability in academic and professional contexts.

Tables

<table>
    <thead>
        <tr>
            <th>Name</th>
            <th>Age</th>
            <th>City</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>John Smith</td>
            <td>25</td>
            <td>Boston</td>
        </tr>
        <tr>
            <td>Jane Doe</td>
            <td>30</td>
            <td>Chicago</td>
        </tr>
    </tbody>
</table>

Forms

<form action="submit.php" method="post">
    <label for="name">Full Name:</label>
    <input type="text" id="name" name="name" required>
    
    <label for="email">Email:</label>
    <input type="email" id="email" name="email" required>
    
    <label for="message">Message:</label>
    <textarea id="message" name="message" rows="4"></textarea>
    
    <label for="newsletter">
        <input type="checkbox" id="newsletter" name="newsletter">
        Subscribe to newsletter
    </label>
    
    <button type="submit">Send Message</button>
</form>

Common Attributes

Attributes provide additional information about HTML elements and control their behavior. Understanding essential attributes enables proper styling, accessibility, and functionality for coursework projects.

Essential Attributes

Attribute Purpose Example
id Unique identifier for element <div id="main-content">
class CSS styling group <p class="highlight">
src Source file location <img src="photo.jpg">
href Link destination <a href="page.html">
alt Alternative text for images <img alt="Description">
title Tooltip text on hover <a title="More info">
lang Language declaration <html lang="en">

Common Beginner Mistakes

Recognizing and avoiding common HTML errors saves debugging time and ensures assignment success. These mistakes frequently appear in beginner code and have straightforward solutions.

Missing Closing Tags

❌ Wrong:

<p>This paragraph is not closed
<div>This div is not closed
<h1>Missing closing tag

✅ Correct:

<p>This paragraph is properly closed</p>
<div>This div is properly closed</div>
<h1>Proper closing tag</h1>

Incorrect Nesting

❌ Wrong:

<p><div>Block elements inside inline</div></p>
<strong><em>Overlapping tags</strong></em>

✅ Correct:

<div><p>Block elements contain inline</p></div>
<strong><em>Properly nested tags</em></strong>

Missing Alt Text

❌ Wrong:

<img src="photo.jpg">
<img src="chart.png" alt="">

✅ Correct:

<img src="photo.jpg" alt="Students working in library">
<img src="chart.png" alt="Sales data showing 20% increase">

Case Sensitivity Errors

❌ Wrong:

<P>Mixed case tags</p>
<IMG SRC="photo.jpg" ALT="description">

✅ Correct:

<p>Consistent lowercase</p>
<img src="photo.jpg" alt="description">

Debugging Tips

  • Validate your HTML: Use the W3C HTML Validator to check for syntax errors
  • Check the browser console: Open Developer Tools (F12) to see error messages
  • Indent your code: Proper indentation makes nesting errors visible
  • Use a code editor: Syntax highlighting catches many mistakes automatically
  • Test in multiple browsers: Ensure consistent behavior across different platforms
